## Task: AI 向任务｜整理 AI 基础概念卡片

#### Agent

LLM 本身只做一件事：给定输入，输出文字。Agent 是在这之上加了一个循环——模型可以决定下一步做什么，执行，看结果，再决定下一步，直到任务完成。

关键在于"自主决策+执行"这个闭环。一个只回答问题的 ChatBot 不是 Agent；一个能自己搜索资料、写代码、跑代码、根据报错修改再跑的系统，才是。

**例子：** 让 Agent 帮你做竞品分析。它可能依次：搜索竞品官网 → 抓取内容 → 整理成表格 → 发现数据缺失 → 自己补充搜索 → 最终输出报告。全程你只给了一句指令。

**使用边界：** Agent 越自主，越难预测它会做什么。任务链越长，中间某一步出错累积的偏差越大。不适合对准确性要求极高、不可逆操作多的场景（比如直接执行交易），除非配合 human-in-the-loop。

---

#### Tool Use（工具调用）

模型本身没有"手"，Tool Use 给了它手。本质是：模型在生成回复时，可以输出一个结构化的"调用请求"（比如 `search("上海天气")`），外部系统执行这个调用，把结果返回给模型，模型再继续生成。

模型自己不执行工具，它只负责"决定调用什么、传什么参数"，真正跑的是外部代码。

**例子：** Claude 回答"今天上海多少度"，它不知道实时天气，但它可以调用一个 `get_weather` 工具，拿到结果再告诉你。

**误区：** Tool Use 不等于 Agent。单次调用一个工具、拿结果、结束，这是 Tool Use，不构成 Agent。Agent 是多轮、自主决策的循环，Tool Use 是 Agent 的能力组件之一。

---

#### AI Coding

泛指用 AI 辅助或自动化编写代码的工作方式，从 Copilot 式的行内补全，到 Cursor 式的多文件编辑，到 Claude Code 式的完整任务执行，粒度差异很大。

**更准确的分类：**
- **补全级：** 你在写，AI 帮你填（Copilot）
- **对话级：** 你描述需求，AI 生成片段，你审核粘贴
- **Agent 级：** 你给任务，AI 自主读取上下文、修改多个文件、验证结果（Claude Code、Devin）

**使用边界：** AI 对自己写的代码会"自信地犯错"——逻辑错误、幻觉 API、边界条件遗漏。粒度越大、自主性越高，需要你 review 的能力要求反而越高，不是替代你思考，是放大你的判断。

---

#### Guardrails（护栏）

对模型的输入或输出加约束，防止它做不该做的事，或输出不符合预期格式/内容的结果。

**护栏可以在不同层：**
- **输入层：** 过滤用户输入里的有害内容、注入攻击
- **输出层：** 检测模型输出是否合规（不含敏感词、格式正确、不乱承诺）
- **逻辑层：** 限制 Agent 能调用哪些工具、能访问哪些资源

**例子：** 金融场景里，模型不能输出"这支股票一定会涨"这类确定性建议，Guardrails 会检测并拦截或改写。

**误区：** Guardrails 不是让模型"更聪明"，是给它套边界。护栏设计不好会误杀正常输出（过严），或拦不住真正的风险（过松）。它是工程问题，不是提示词问题。

---

#### Tracing（追踪）

Agent 或复杂 LLM 应用跑起来之后，你看到的只是最终输出。Tracing 是记录"中间发生了什么"的机制——模型调用了哪些工具、传了什么参数、每一步耗时多少、哪一步出了错。

**类比：** 分布式系统里的链路追踪（OpenTelemetry），只不过追踪对象从微服务变成了 LLM 调用链。

**例子：** 一个 Agent 任务跑了 2 分钟，最终输出是错的。没有 Tracing，你不知道哪步出了问题；有 Tracing，你能看到"第 3 步搜索返回了空结果，第 4 步模型开始幻觉"。

**使用边界：** 本地跑个小脚本不需要 Tracing；一旦系统进入生产、任务链超过 3 步、或者要做性能优化和 debug，Tracing 就是必需品。常见工具：LangSmith、Langfuse、Arize Phoenix。

---

#### Human-in-the-loop（人工介入）

在 Agent 的自动化流程中，设置检查点，让人在关键决策前确认或干预，而不是全程放手让模型跑。

不是"AI 做不到所以人来做"，是主动设计的风险控制机制。

**介入粒度：**
- **审批式：** Agent 要执行高风险操作前弹出确认
- **纠偏式：** Agent 跑完一段后给你 summary，你可以说"这个方向不对，重来"
- **监控式：** 全程自动，但异常时触发人工介入

**使用边界：** 介入点设太多，自动化意义就没了；设太少，Agent 出错代价很高。设计原则是：在不可逆操作和高风险决策前设检查点，其余放行。
